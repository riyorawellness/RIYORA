## Referral flow — end-to-end walkthrough

1. **Signup with a referral ID** → `POST /api/auth/firebase/register` validates the referral matches pattern `^RW\d{6}$`, looks up sponsor in `memberships`, rejects on miss.
2. **Sponsor mapping saved** → `users.sponsor_membership_id` + `memberships.sponsor_membership_id` + `referral_tree` node inserted with `level = sponsor.level + 1`.
3. **Program purchase** (one-time or subscription renewal) → `commission_engine.create_commissions_for_purchase()`:
   - Walks up the tree at most 3 levels (L1 = direct sponsor, L2 = sponsor's sponsor, L3 = sponsor²).
   - Skips the company root `RW000000` at every level (never receives commissions).
   - Applies the 4-session activity gate — sponsors below threshold get `status='rejected'` with reason.
   - Amount = `program.total * (rate or fixed)` at each level. Idempotent via unique `(purchase_id, sponsor_membership_id)`.
4. **Wallet** = sum of commissions grouped by `status` per sponsor (`services.commission_engine.summarise_user`).
5. **Redemption** → admin uses `/admin/payouts` to select approved commissions per sponsor, mark payout paid with method + reference, which moves those commissions to `status='paid'` and locks them.
