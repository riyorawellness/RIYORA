# RIYORA Wellness — Production Deployment Runbook

**Target platform**: Any Ubuntu 22.04+ VPS (Hostinger, DigitalOcean, Linode, Hetzner, AWS EC2, ...).
**Estimated time**: 15 min first-time, 2 min for subsequent deploys.
**Prerequisites**: Root/sudo SSH access, a domain that points to your VPS's public IPv4.

---

## 📁 What lives in this folder

```
deploy/
├── docker-compose.yml           # 4 services: mongo + backend + frontend + nginx (+ certbot)
├── .env.example                 # Every environment variable you need to fill in
├── backend/Dockerfile           # Python 3.11 + FastAPI + Firebase Admin
├── frontend/
│   ├── Dockerfile               # Multi-stage: node build → nginx serve
│   └── nginx-spa.conf           # SPA fallback + gzip + PWA-friendly caching
├── nginx/
│   └── default.conf             # Outer TLS-terminating reverse proxy + rate limiting
├── scripts/
│   ├── deploy.sh                # One-command turnkey deploy
│   ├── verify.sh                # 10 post-deploy smoke tests
│   ├── rollback.sh              # Flip back to previous image
│   └── certbot-init.sh          # First-time Let's Encrypt bootstrap
└── DEPLOYMENT.md                # This file
```

## 🖥️ Step 1 — One-time VPS setup (5 min)

SSH into your VPS as root, then:

```bash
# 1. Update & install docker + git
apt update && apt upgrade -y
curl -fsSL https://get.docker.com | bash
apt install -y docker-compose-plugin git

# 2. Clone your repo (or pull if already cloned)
cd /opt
git clone https://github.com/YOUR-USER/YOUR-REPO.git riyora
cd riyora

# 3. Point DNS
# In your DNS panel, create an A record:
#   Host:  app      (or @ for root domain)
#   Type:  A
#   Value: <VPS public IP>
# Wait ~5 min for propagation. Verify with:  dig +short app.YOURDOMAIN.com
```

---

## 🔐 Step 2 — Configure secrets (5 min, one-time)

```bash
cd /opt/riyora/deploy

# 1. Copy the template
cp .env.example .env
nano .env    # fill in every CHANGE_ME_* placeholder

# 2. Generate a strong JWT secret
sed -i "s|JWT_SECRET=.*|JWT_SECRET=$(openssl rand -hex 64)|" .env

# 3. Upload the Firebase service account JSON
#    On your LOCAL machine, run:
#      scp riyora-8059e-firebase-adminsdk-*.json \
#          root@YOUR-VPS-IP:/opt/riyora/backend/firebase-admin.json
#    Then on the VPS:
chmod 600 /opt/riyora/backend/firebase-admin.json
```

### Values you MUST replace in `.env`

| Key | Value |
|---|---|
| `DOMAIN` | `app.riyorawellness.com` (whatever your production hostname is) |
| `LETSENCRYPT_EMAIL` | your ops email — Let's Encrypt sends renewal warnings here |
| `REACT_APP_BACKEND_URL` | `https://<DOMAIN>` |
| `CORS_ORIGINS` | `https://<DOMAIN>` |
| `MONGO_ROOT_PASSWORD` | 16+ char strong random string |
| `JWT_SECRET` | `openssl rand -hex 64` (auto-generated by step 2.2 above) |
| `ADMIN_PASSWORD` | Your admin password — change immediately after first login |
| `RAZORPAY_WEBHOOK_SECRET` | Razorpay dashboard → Settings → Webhooks → shown once |

Firebase, Razorpay and Firebase Web SDK values in `.env.example` are already pre-filled from your project.

## 🔥 Step 3 — Firebase Console configuration (2 min, one-time)

Before the first deploy, register your production domain with Firebase Auth or Google sign-in will fail.

1. Open https://console.firebase.google.com/ → project **riyora-8059e**
2. **Authentication** → **Settings** → **Authorized domains** → **Add domain**
3. Add your production domain (e.g. `app.riyorawellness.com`)
4. If you use both root and www, add both

> Localhost + `*.firebaseapp.com` are already whitelisted by default.

---

## 💳 Step 4 — Razorpay webhook (2 min, one-time)

1. Razorpay Dashboard → **Settings** → **Webhooks** → **Add New Webhook**
2. **Webhook URL**: `https://<YOUR-DOMAIN>/api/payments/razorpay/webhook`
3. **Alert email**: same as `LETSENCRYPT_EMAIL`
4. **Events**: check `payment.captured`, `payment.failed`, `order.paid`, `subscription.charged`, `subscription.cancelled`
5. Copy the **Webhook Secret** shown once → paste into `.env`:
   ```env
   RAZORPAY_WEBHOOK_SECRET=<paste-here>
   ```

## 🚀 Step 5 — First deploy

```bash
cd /opt/riyora/deploy
chmod +x scripts/*.sh
./scripts/deploy.sh
```

The script does everything:
1. Sanity-checks `.env` + firebase JSON exist
2. Pulls latest git
3. Substitutes your DOMAIN into `nginx/default.conf`
4. If no TLS cert yet → runs Certbot to issue one
5. Snapshots current image digests (for rollback)
6. Builds backend (`docker compose build backend`)
7. Builds frontend with `--no-cache` so `REACT_APP_*` env is baked fresh
8. Runs `docker compose up -d`
9. Waits for backend health
10. Runs `verify.sh` (10 tests) — if any fail, **automatic rollback**

Expected output on success:
```
✔ backend /api/health returns 200
✔ backend /api/health/live returns 200
✔ frontend homepage returns 200 with <div id="root">
✔ Firebase apiKey present in JS bundle
✔ admin login returns access_token
✔ firebase/sync endpoint alive
✔ BRV overall PASS
✔ Razorpay reports LIVE mode
✔ Firebase Admin SDK reports LIVE
✔ OTP endpoints removed
─────────────────────────
  Passed: 10 · Failed: 0
─────────────────────────

================================================
  RIYORA is live at https://app.riyorawellness.com
================================================
```

## 🔄 Step 6 — Subsequent deploys (2 min)

Every time you `git push` to `main`:

```bash
ssh root@YOUR-VPS
cd /opt/riyora/deploy
./scripts/deploy.sh
```

That's it. Zero-downtime if backend is happy, automatic rollback if not.

---

## 🩹 Manual rollback

If you deployed a bad version manually or `verify.sh` reported false-negatives:

```bash
cd /opt/riyora/deploy
./scripts/rollback.sh
```

The last-good image digests are stored in `.rollback/previous.env` after each successful deploy.

## 🛡️ Post-launch — first-day checklist

Do these WITHIN the first hour after go-live:

- [ ] **Change admin password** — log into `/admin/login`, then Profile → Change password (or `POST /api/admin/reset-password-self`)
- [ ] **Test one real registration** — open `/register` in incognito → Continue with Google → complete profile → land on `/app/home`
- [ ] **Test one real Razorpay purchase** — pick your ₹1 test product, complete checkout with a real card, confirm invoice PDF downloads
- [ ] **Verify webhook events** — `/admin/qa/live-check` → Recent Razorpay webhook events should show the payment
- [ ] **Set up nightly Mongo backups** — see `docs/backups.md` (or add to your existing cron: `0 3 * * * docker exec riyora-mongo mongodump --archive=/backups/nightly-$(date +\%F).archive.gz --gzip`)
- [ ] **Set up an uptime monitor** — UptimeRobot free plan hitting `https://<DOMAIN>/api/health` every 5 minutes
- [ ] **Enable Firebase billing** — if you expect >10K logins/day, add a billing account to Firebase (free tier covers most small apps)

## 🐛 Troubleshooting

### "changes not showing after push"
Almost always a stale frontend build. The container serves the OLD bundle until you rebuild:
```bash
docker compose build --no-cache frontend
docker compose up -d frontend
```
Then in your browser: **DevTools → Application → Service Workers → Unregister → hard-refresh** (Ctrl+Shift+R).

### "Google Sign-In popup fails with `auth/unauthorized-domain`"
You forgot **Step 3**. Add your domain to Firebase Console → Authentication → Settings → Authorized domains.

### "TLS certificate not issued"
DNS not propagated yet. Verify with `dig +short app.YOURDOMAIN.com` → should return your VPS IP. Then re-run:
```bash
rm -rf certbot/conf/live/*
./scripts/deploy.sh
```

### "Razorpay webhook signature invalid" in `/admin/qa/live-check` webhook feed
`RAZORPAY_WEBHOOK_SECRET` in `.env` doesn't match what you set in Razorpay dashboard. Copy the exact value from Razorpay Settings → Webhooks → your endpoint → **Show secret**.

### Backend keeps restarting
```bash
docker compose logs backend --tail=100
```
Common culprits:
- `firebase-admin.json` missing at `/app/firebase-admin.json` inside container (upload it)
- Mongo not reachable (check `MONGO_ROOT_PASSWORD` matches what's in the Mongo container)
- `JWT_SECRET` unset (backend refuses to start without one)

### Frontend loads but Firebase login button does nothing
Open DevTools console. If you see `Firebase: Error (auth/invalid-api-key)` — the `REACT_APP_FIREBASE_API_KEY` build arg wasn't baked in. Rebuild:
```bash
docker compose build --no-cache frontend
docker compose up -d frontend
```
